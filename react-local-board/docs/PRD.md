# React Local Board PRD

## 1. 개요

React와 TypeScript로 브라우저 `localStorage` 기반 게시판을 구현한다. 사용자는 백엔드나 로그인 없이 게시글을 작성, 조회, 수정, 삭제할 수 있으며, 새로고침 후에도 데이터가 유지되어야 한다.

## 2. 목표

- React Router를 사용해 목록, 상세, 작성, 수정 화면을 URL 기준으로 분리한다.
- 게시글 목록, 상세, 작성, 수정, 삭제 기능을 완성한다.
- 제목 검색과 최신순 정렬을 제공한다.
- 게시글 데이터는 `localStorage`에 저장하고 새로고침 후에도 유지한다.
- 잘못된 입력, 깨진 저장 데이터, 저장소 접근 오류를 안전하게 처리한다.
- 핵심 로직은 테스트 가능하게 분리한다.

## 3. 범위

- 게시글 목록 화면
- 게시글 상세 화면

- 게시글 삭제 기능
- 제목 기반 검색
- 최신순 정렬
- 상세 진입 시 조회수 증가
- `localStorage` 기반 영속 저장
- 입력값 검증
- 저장소 오류 처리
- 핵심 로직 테스트
- 실행법과 프로젝트 구조 문서화

## 4. 비범위

- 로그인, 회원가입, 인증, 권한

- 관리자 기능
- 배포 자동화

## 5. 게시글 데이터

게시글은 다음 필드를 가진다.

```ts
type Post = {
  id: string
  title: string
  content: string
  createdAt: string
  updatedAt: string
  viewCount: number
}
```

- `title`: 게시글 제목
- `content`: 게시글 내용
- `createdAt`: 작성일
- `updatedAt`: 수정일
- `viewCount`: 조회수
- `id`: 게시글 식별용 고유 문자열

조회수는 목록에서 상세 페이지로 들어갈 때마다 1 증가한다.

## 6. 사용자 흐름

1. 사용자는 게시글 목록에 진입한다.
2. 사용자는 제목으로 게시글을 검색할 수 있다.
3. 사용자는 최신순으로 정렬된 게시글 목록을 확인한다.
4. 사용자는 새 글 작성 화면에서 제목과 내용을 입력해 게시글을 생성한다.
5. 사용자는 목록에서 게시글을 선택해 상세 화면으로 이동한다.
6. 상세 화면에 진입할 때 해당 게시글의 조회수가 1 증가한다.
7. 사용자는 상세 화면에서 게시글을 수정하거나 삭제할 수 있다.
8. 사용자는 페이지를 새로고침해도 기존 게시글 데이터를 다시 확인할 수 있다.

## 7. 기능 요구사항

### 7.1 목록

- 게시글 목록에는 제목, 작성일, 수정일, 조회수를 표시한다.
- 목록은 기본적으로 최신순으로 정렬한다.
- 최신순 기준은 `createdAt` 내림차순이다.
- 검색어가 없으면 전체 게시글을 표시한다.
- 검색어가 있으면 제목에 검색어가 포함된 게시글만 표시한다.
- 검색은 대소문자 차이에 과도하게 의존하지 않도록 처리한다.

### 7.2 상세

- 상세 화면에는 제목, 내용, 작성일, 수정일, 조회수를 표시한다.
- 목록에서 상세 화면으로 진입할 때마다 조회수가 1 증가한다.
- 존재하지 않는 게시글 ID로 접근하면 사용자가 목록으로 돌아갈 수 있는 안내를 제공한다.

### 7.3 작성

- 사용자는 제목과 내용을 입력해 게시글을 작성할 수 있다.
- 제목과 내용은 키보드 입력 중 화면 오류 없이 즉시 폼 상태에 반영되어야 한다.
- 새 게시글의 조회수는 0으로 시작한다.
- 작성 성공 후 사용자는 새 게시글 상세 화면으로 이동한다.

### 7.4 수정

- 사용자는 기존 게시글의 제목과 내용을 수정할 수 있다.
- 제목과 내용은 키보드 입력 중 화면 오류 없이 즉시 폼 상태에 반영되어야 한다.
- 수정 시 `updatedAt`을 갱신한다.
- 수정은 기존 `createdAt`과 `viewCount`를 유지한다.
- 존재하지 않는 게시글을 수정하려는 경우 오류 상태를 보여주고 목록으로 돌아갈 수 있게 한다.

### 7.5 삭제

- 사용자는 게시글을 삭제할 수 있다.
- 삭제 전 확인 절차를 제공한다.
- 삭제 성공 후 목록 화면으로 이동한다.

## 8. 검증 및 오류 처리

- 제목은 필수값이다.
- 내용은 필수값이다.
- 제목과 내용은 앞뒤 공백을 제거한 뒤 검증한다.
- 공백만 입력된 제목이나 내용은 허용하지 않는다.
- 작성/수정 폼에서 입력 이벤트 처리 중 런타임 오류가 발생해 화면 전체가 비는 상태가 없어야 한다.
- `localStorage`에서 데이터를 읽을 때 JSON 파싱 오류를 처리한다.
- 저장된 데이터가 기대한 배열 또는 게시글 구조가 아니면 안전한 기본값으로 복구한다.
- `localStorage` 쓰기 실패 시 사용자에게 저장 실패 상태를 알린다.
- 오류가 발생해도 화면 전체가 중단되지 않아야 한다.

## 9. 라우팅 요구사항

React Router를 사용한다.

- `/` 또는 `/posts`: 게시글 목록
- `/posts/new`: 게시글 작성
- `/posts/:postId`: 게시글 상세
- `/posts/:postId/edit`: 게시글 수정

정확한 경로명은 구현 단계에서 조정할 수 있으나, 목록, 상세, 작성, 수정 화면은 URL로 구분되어야 한다.

## 11. 완료 조건

- 게시글 목록, 상세, 작성, 수정, 삭제가 동작한다.
- 제목 검색이 동작한다.
- 게시글 목록이 최신순으로 정렬된다.
- 상세 진입 시 조회수가 1 증가한다.
- 새로고침 후에도 게시글 데이터가 유지된다.
- React Router로 주요 화면이 분리되어 있다.
- 백엔드와 로그인 기능이 없다.
- 입력값 검증과 `localStorage` 오류 처리가 구현되어 있다.
- 작성/수정 폼에서 제목과 내용 입력 시 화면이 유지된다.
- `npm run build`가 성공한다.
- 핵심 로직 테스트가 통과한다.
- `README.md`에 실행법과 구조가 기록되어 있다.

## 12. 결정된 사항

- 게시글 데이터 필드는 제목, 내용, 작성일, 수정일, 조회수를 포함한다.
- 조회수는 목록에서 상세 페이지로 들어갈 때마다 1 증가한다.
- 작성 성공 후 이동 위치는 새 게시글 상세 화면으로 고정한다.
- 작성/수정 폼 입력 이벤트는 이벤트 객체를 비동기 state updater 내부에서 직접 참조하지 않는다.
- 백엔드와 로그인은 이 프로젝트 범위에서 제외한다.
- PRD의 기준 내용은 이 문서를 따른다.
