# 알려진 문제

이 문서는 현재 구현 범위에서 의도적으로 남겨둔 제한, 환경 이슈, 향후 검토 항목을 기록한다.

## KI-001 localStorage 단일 브라우저 저장 한계

- 상태: 알려진 제한
- 영향:
  - 게시글 데이터는 현재 브라우저의 `localStorage`에만 저장된다.
  - 다른 브라우저, 다른 기기, 다른 사용자와 데이터가 동기화되지 않는다.
- 이유:
  - PRD의 비범위에 백엔드 API, 로그인, 다중 사용자 동기화가 포함되어 있다.
- 대응:
  - 현재 범위에서는 조치하지 않는다.
  - 동기화가 필요해지면 백엔드 저장소, 인증, 데이터 마이그레이션 요구사항을 새 PRD 범위로 정의한다.
- 연결 Task:
  - T010

## KI-002 깨진 localStorage 데이터는 빈 목록으로 복구됨

- 상태: 알려진 동작
- 영향:
  - 저장된 JSON이 깨졌거나 스키마가 맞지 않으면 앱은 빈 게시글 목록으로 복구한다.
  - 손상된 기존 데이터 자체를 자동으로 복원하지는 않는다.
- 이유:
  - 데이터 무결성이 확인되지 않은 payload를 계속 사용하는 것보다 안전한 기본값으로 복구하는 편이 앱 안정성에 유리하다.
- 대응:
  - 현재 구현은 오류 메시지를 표시하고 빈 배열로 복구한다.
  - 향후 필요하면 손상 데이터 백업, 복구 안내, export/import 기능을 별도 Task로 정의한다.
- 연결 Task:
  - T002
  - T008
  - T010

## KI-003 Codex Windows 셸 Node/npm 환경 변수 이슈

- 상태: 환경 이슈
- 영향:
  - Codex Windows 셸에서 `SystemRoot`, `windir`, `ComSpec`, `APPDATA`, `LOCALAPPDATA` 등 핵심 환경 변수가 누락되면 `node` 또는 `npm` 실행 시 `ncrypto::CSPRNG(nullptr, 0)` 오류가 발생할 수 있다.
- 이유:
  - 프로젝트 코드 문제가 아니라 실행 셸 환경 문제다.
- 대응:
  - 검증 명령 실행 전 필요한 환경 변수를 현재 셸에 명시해 우회한다.
  - README에 우회 방법을 기록했다.
- 연결 Task:
  - T001
