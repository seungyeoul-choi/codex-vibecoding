# 프로젝트 에이전트 지침

이 파일은 `react-local-board` 프로젝트에서 Codex/OMX 에이전트가 따라야 하는 최상위 작업 계약이다. 모든 응답, 진행 상황, 문서 작성은 한국어로 작성한다.

## 프로젝트 목표

React와 TypeScript로 `localStorage` 기반 게시판을 완성한다.

## 완료 조건

- 게시글 목록, 상세, 작성, 수정, 삭제가 동작한다.
- 제목 검색과 최신순 정렬이 동작한다.
- 새로고침 후에도 데이터가 유지된다.
- 데이터 검증과 `localStorage` 오류를 처리한다.
- `npm run build`가 성공한다.
- 핵심 로직 테스트가 통과한다.
- `README.md`에 실행법과 구조를 기록한다.

## 작업 원칙

- 가능한 경우 한 명의 에이전트가 직접 읽고, 수정하고, 검증한다.
- 변경 전에는 현재 코드 구조와 기존 패턴을 먼저 확인한다.
- 사용자 변경사항을 되돌리지 않는다. 작업 중 발견한 기존 변경은 사용자 작업으로 간주하고 보존한다.
- 변경 범위는 요청과 완료 조건에 필요한 파일로 제한한다.
- 새 의존성은 명확히 필요할 때만 추가한다.
- 구현보다 삭제, 기존 유틸리티 재사용, 단순한 구조를 우선한다.
- 완료를 주장하기 전에 반드시 검증 결과를 확인한다.

## 구현 방향

- React + TypeScript + Vite 구조를 유지한다.
- 게시판 데이터는 브라우저 `localStorage`에 저장한다.
- 저장 데이터는 새로고침 후에도 유지되어야 한다.
- 데이터 읽기/쓰기 과정에서 JSON 파싱 실패, 저장소 접근 실패, 잘못된 데이터 형태를 처리한다.
- 사용자 입력은 제목과 내용의 필수값, 공백 문자열, 길이 등 핵심 검증을 거친다.
- 목록 화면에서는 제목 검색과 최신순 정렬을 제공한다.
- 작성, 상세, 수정, 삭제 흐름은 사용자가 자연스럽게 오갈 수 있게 구성한다.
- UI는 과한 장식보다 읽기 쉬운 정보 구조와 반복 사용에 편한 조작을 우선한다.

## OMX 사용 규칙

- OMX는 이 프로젝트의 작업 조율 계층이다.
- 명시적인 `$name` 호출이 있으면 해당 워크플로 스킬을 먼저 확인하고 따른다.
- `/skills` 요청이 있으면 사용 가능한 스킬 탐색을 돕는다.
- 런타임 마커는 비파괴적으로 유지한다.
  - `<!-- OMX:RUNTIME:START --> ... <!-- OMX:RUNTIME:END -->`
  - `<!-- OMX:TEAM:WORKER:START --> ... <!-- OMX:TEAM:WORKER:END -->`
- 설치 또는 상태 점검이 필요하면 `omx setup`, `omx doctor`를 사용할 수 있다.

## 역할 라우팅

- 일반 구현은 직접 수행한다.
- 저장소 구조 탐색이나 심볼 찾기는 `explore` 역할이 적합하다.
- 외부 공식 문서 확인이 필요한 경우 `researcher` 역할이 적합하다.

- 서브에이전트는 작업이 독립적이고 병렬화 이점이 분명할 때만 사용한다.
